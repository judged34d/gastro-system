Cloudflare Tunnel – Anleitung für Gastro-System

Mit einem Named Tunnel erreichst du das System von außen über HTTPS, ohne Portfreigabe am Router.

1. Voraussetzungen

  1. Domain bei Cloudflare (Nameserver zeigen auf Cloudflare).
  2. cloudflared.exe installiert (Standard: C:\Program Files\cloudflared\cloudflared.exe).
  3. Gastro-System auf diesem PC installiert (Backend 8000, Frontend 8081).

2. Tunnel in Cloudflare anlegen

  1. Einloggen: Cloudflare Zero Trust (oder Dashboard → Zero Trust).
  2. Networks → Tunnels → Create a tunnel.
  3. Typ: Cloudflared, Name z. B. gastro-system.
  4. Beim Schritt „Install connector“ die Tunnel-ID notieren und die Credentials-JSON herunterladen (Datei wie <tunnel-uuid>.json).

3. Credentials auf diesem PC

Die JSON-Datei gehört nach:

%USERPROFILE%\.cloudflared\<TUNNEL-UUID>.json

Im Setup gibst du diese Datei an – sie wird dorthin kopiert und in cloudflared\config.yml referenziert.

4. Öffentliche Hostnames (im Tunnel)

Unter Public Hostnames zwei Einträge anlegen:

Subdomain / HostnameServiceZweck
app.deine-domain.dehttp://127.0.0.1:8081Bedienoberfläche (Frontend)
api.deine-domain.dehttp://127.0.0.1:8000Backend-API

Die exakt gleichen Hostnames trägst du im Setup ein.

5. DNS bei der Domain

Cloudflare legt oft automatisch CNAME-Einträge an. Falls nicht, manuell in DNS → Records:

TypNameZiel
CNAMEapp<TUNNEL-UUID>.cfargotunnel.com
CNAMEapi<TUNNEL-UUID>.cfargotunnel.com
Proxy-Status (orange Wolke) kann bei Tunnel-Hostnames „Proxied“ bleiben – Cloudflare terminiert TLS.

6. Frontend / API-URL

Die App erkennt *.deine-domain.de automatisch, wenn die Domain im Browser-Host steht. Für die API wird typischerweise https://api.deine-domain.de genutzt (siehe frontend/api_base.js – eigene Domain ggf. dort anpassen).

7. Test

  1. Im Gastro Control Panel: Tunnel verbinden oder run-cloudflared.cmd.
  2. Im Tunnel-Dashboard: Status Healthy.
  3. Browser: https://app.deine-domain.de und https://api.deine-domain.de/health.

8. Ohne Tunnel (nur LAN)

Cloudflare im Setup deaktivieren – Zugriff nur über die ermittelte LAN-IP, z. B. http://192.168.x.x:8081.
